# 포트폴리오 비교 리뷰어 - 프로젝트 계획서

## 개요

- **주제**: AI 기반 개발자 포트폴리오 정형화 및 비교 시스템
- **LLM**: Upstage Solar API
- **구조**: Frontend + Backend (웹 애플리케이션)

---

## 사용자 입력

| 구분 | 항목 |
|------|------|
| 필수 | 지원자 포트폴리오 다수 (PDF / URL) |
| 필수 | 정형화된 서식 (어떤 항목으로 정리할지) |
| 필수 | 필요 스펙 (요구 기술 스택, 경력 조건 등) |

---

## 전체 흐름

```
사용자 입력 (포트폴리오 + 서식 + 필요 스펙)
      ↓
① Solar: 포트폴리오에서 정보 추출
         (이름, 기술스택, 경력, 프로젝트, 학력 등)
      ↓
② 해시 테이블: 필요 스펙 키워드 저장 및 O(1) 매칭 탐지
      ↓
③ Edit Distance (DP): 기술 스택 유사도 계산
   (예: "ReactJS" ↔ "React.js" → 같은 기술로 처리)
      ↓
④ LCS (DP): 필요 스펙 vs 지원자 스펙 공통 항목 계산
   → 매칭 점수 산출
      ↓
⑤ 정렬: 매칭 점수 기준 지원자 순위 정렬
      ↓
⑥ BST: 특정 키워드로 지원자 빠르게 검색/필터링
      ↓
결과 화면 표시
(정형화된 서식으로 수평 나열 + 하이라이트 + 원본 링크 버튼)
```

## 알고리즘별 예시

### ① Solar
```
입력: 지원자 포트폴리오 PDF / GitHub URL / 개인 사이트 URL

출력 (정형화된 서식):
{
  "이름": "홍길동",
  "기술스택": ["React", "Python", "Docker", "AWS"],
  "경력": "3년",
  "프로젝트": [
    {"이름": "쇼핑몰 플랫폼", "기간": "6개월", "역할": "프론트엔드 개발"},
    {"이름": "챗봇 서비스", "기간": "3개월", "역할": "백엔드 개발"}
  ],
  "학력": "OO대학교 컴퓨터공학과",
  "포트폴리오링크": "https://github.com/honggildong"
}
```
→ 제각각인 포트폴리오를 통일된 서식으로 자동 정리

---

### ② 해시 테이블 (Hash Table)
```
필요 스펙: ["React", "Python", "Docker"]

해시셋 구성 (O(1) 조회):
requiredSpecs = {"React", "Python", "Docker"}

지원자A 기술스택: ["React", "Vue", "Python", "Git"]

매칭 확인:
"React"  → requiredSpecs에 있음 → O(1) → 🟡 하이라이트
"Vue"    → requiredSpecs에 없음 → O(1) → 표시 없음
"Python" → requiredSpecs에 있음 → O(1) → 🟡 하이라이트
"Git"    → requiredSpecs에 없음 → O(1) → 표시 없음

→ 지원자 수 / 스펙 수가 많아져도 매칭 속도 일정 (O(1))
→ UI에서 매칭된 키워드가 강조되어 표시됨
```

---

### ③ Edit Distance (DP)
```
문제: 기술명이 사람마다 다르게 표기됨

필요 스펙: "React"

지원자A: "React"   → 거리 0 → 매칭 O
지원자B: "ReactJS" → 거리 2 → 임계값 이하 → 매칭 O
지원자C: "React.js"→ 거리 3 → 임계값 이하 → 매칭 O
지원자D: "Angular" → 거리 7 → 임계값 초과 → 매칭 X

→ 표기 방식이 달라도 같은 기술로 인식 가능
```

---

### ④ LCS (DP)
```
필요 스펙:  [React, Python, Docker, AWS, Git]
지원자A 스펙: [React, Vue, Python, Git, Linux]

LCS 계산:
공통: React, Python, Git (길이 3)

매칭 점수 = 공통 항목 수 / 필요 스펙 수
           = 3 / 5 = 60%

→ 지원자별 매칭 점수 산출
→ 화면에 "매칭률 60%" 표시
```

---

### ⑤ 정렬
```
매칭 점수 높은 순:
지원자C (80%) → 지원자A (60%) → 지원자B (40%) → 지원자D (20%)

경력 많은 순:
지원자B (5년) → 지원자A (3년) → 지원자C (2년) → 지원자D (1년)

→ 인사담당자가 원하는 기준으로 정렬 전환 가능
```

---

### ⑥ BST
```
인사담당자가 특정 키워드로 검색할 때:

"Docker" 입력
→ BST 탐색 O(log n)
→ Docker 보유 지원자만 빠르게 필터링

"Python" 입력
→ Python 보유 지원자만 즉시 조회

→ 지원자 수가 많아져도 빠른 검색 유지
```
