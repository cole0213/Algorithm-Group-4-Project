프로젝트 1: 실시간 추천 피처 파이프라인
------------------------------------------

카카오스타일 내부 프로젝트.

배경:
  기존 배치 기반 추천 피처가 하루 1번 업데이트되어 실시간성이 부족했습니다.
  사용자가 방금 본 상품이 추천에 반영되지 않는 문제가 있었어요.

아키텍처:

  [사용자 행동] -> [Kafka] -> [Flink] -> [Feature Store (Redis)]
                                    \-> [BigQuery] (분석용 백업)

구현:
  - Kafka Connect로 앱 이벤트 수집 설정
  - Flink 스트리밍 잡으로 세션 피처 실시간 계산
  - Redis에 ttl 설정하여 피처 자동 만료 처리
  - Feast(Feature Store 라이브러리) 도입으로 학습/서빙 피처 일관성 확보

결과:
  CTR 4.2% -> 5.8% 향상 (추천팀 측정 결과)
  피처 staleness 24시간 -> 30초 이하


프로젝트 2: LLM 기반 상품 태그 자동 생성 (사이드 프로젝트)
------------------------------------------

쇼핑몰 크롤링 데이터에 상품 카테고리/스타일 태그를 자동으로 달아주는 시스템.

개인적으로 LLM 활용법 공부하려고 만들었어요.

구성:
  - 상품 이미지 + 설명 텍스트를 Claude API에 전달
  - 응답을 파싱해서 structured tag 추출
  - 결과를 PostgreSQL에 저장, FastAPI로 조회 API 제공

배운 것:
  LLM 응답이 항상 일정하지 않아서 파싱 로직이 계속 깨졌어요.
  function calling / structured output을 쓰니 훨씬 안정적이 됐습니다.

코드 스니펫 (핵심 부분):

```python
from anthropic import Anthropic

client = Anthropic()

def extract_product_tags(image_url: str, description: str) -> dict:
    response = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=512,
        tools=[{
            "name": "tag_product",
            "description": "상품에 태그를 달아주세요",
            "input_schema": {
                "type": "object",
                "properties": {
                    "categories": {"type": "array", "items": {"type": "string"}},
                    "style_tags": {"type": "array", "items": {"type": "string"}},
                    "season": {"type": "string", "enum": ["spring", "summer", "fall", "winter", "all"]},
                },
                "required": ["categories", "style_tags", "season"]
            }
        }],
        tool_choice={"type": "tool", "name": "tag_product"},
        messages=[{
            "role": "user",
            "content": [
                {"type": "image", "source": {"type": "url", "url": image_url}},
                {"type": "text", "text": description}
            ]
        }]
    )
    return response.content[0].input
```
