## 🚀 프로젝트

### 1️⃣ Wavey UI — 디자인 시스템

![Wavey UI 컴포넌트 쇼케이스](https://images.unsplash.com/photo-1558655146-d09347e92766?w=800)

![Storybook 화면](https://images.unsplash.com/photo-1633356122544-f134324a6cee?w=800)

> 🔗 [Storybook 문서](https://wavey-ui.chromatic.com) | [GitHub](https://github.com/sohee-uxdev/wavey-ui)

**📌 배경**

입사했을 때 컴포넌트가 팀마다 제각각이었어요. 버튼 하나도 프로젝트마다 색이 달랐고, 동일한 UI를 서로 다르게 구현해서 유지보수 비용이 컸습니다.

**📌 내가 한 것**

1. 기존 프로덕트 UI 감사(Audit) — 중복 컴포넌트 117개 발견
2. Atomic Design 기반 컴포넌트 계층 설계
3. Figma Variables로 토큰 정의 → Style Dictionary로 CSS/JS 변환 자동화
4. 각 컴포넌트에 접근성 속성(ARIA) 기본 포함

**📌 주요 컴포넌트 목록 (일부)**

| 카테고리 | 컴포넌트 |
| --- | --- |
| Atoms | Button, Input, Badge, Avatar, Icon, Spinner |
| Molecules | Modal, Toast, Tooltip, Dropdown, DatePicker |
| Organisms | DataTable, Form, Navigation, Sidebar |
| Layouts | Container, Grid, Stack, Flex |

**📌 결과**

- 새 프로젝트 UI 개발 속도 약 40% 향상 (팀 피드백 기준)
- 디자이너-개발자 핸드오프 커뮤니케이션 감소
- 사내 5개 프로덕트에서 사용 중

게임 회사 채용 페이지를 리뉴얼했어요. "지원자들이 지원 버튼 누르기 전에 회사를 느낄 수 있으면 좋겠다"는 요청이었어요.

완성하고 나서 클라이언트 분이 눈물 글썽이셨어요. 그게 기억에 남아요.
