[프로젝트 1]  비대면 진료 예약 시스템 (오케어헬스)
─────────────────────────────────────────────────

문제 정의:
  진료 예약 시 의사 스케줄 충돌, 동시 예약 처리 실패 등 데이터 정합성 문제가
  서비스 초기부터 지속적으로 발생하고 있었음.

설계 방향:
  Saga 패턴 기반의 분산 트랜잭션으로 예약 생성-결제-확정 흐름을 관리.
  Redis 기반 분산 락으로 동일 시간대 중복 예약 원천 차단.

핵심 코드 발췌:

```java
@Service
@RequiredArgsConstructor
public class AppointmentSagaOrchestrator {

    private final AvailabilityService availabilityService;
    private final PaymentService paymentService;
    private final NotificationService notificationService;
    private final SagaLogRepository sagaLogRepository;

    @Transactional
    public AppointmentResult execute(AppointmentCommand cmd) {
        String sagaId = UUID.randomUUID().toString();
        SagaLog log = sagaLogRepository.create(sagaId, cmd);

        try {
            SlotReservation slot = availabilityService.hold(cmd.getSlotId());
            log.step("SLOT_HELD");

            PaymentResult payment = paymentService.charge(cmd.getPaymentInfo());
            log.step("PAYMENT_DONE");

            Appointment appt = confirmAppointment(slot, payment);
            notificationService.sendConfirmation(appt);
            log.complete();

            return AppointmentResult.success(appt);

        } catch (Exception e) {
            log.fail(e);
            compensate(log);
            throw new AppointmentFailedException(e);
        }
    }
}
```

결과:
  · 예약 처리 성공률 94.2% → 99.7%
  · 중복 예약 발생 건수 0건 유지 (도입 후 14개월)
  · 고객 CS 인입 예약 관련 60% 감소
