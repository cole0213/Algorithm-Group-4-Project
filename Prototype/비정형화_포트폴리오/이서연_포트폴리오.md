## PROJECTS

### 분산 결제 처리 엔진 (PayFlow 사내 프로젝트)

결제 요청 폭증 시에도 일관된 처리 보장이 필요했던 프로젝트.

**설계 고민 포인트**

기존 시스템은 DB 수준의 row-level lock에 의존하고 있어서, 트래픽 급증 시 lock contention이 심각했습니다. 이를 해결하기 위해 Redis 기반 분산 락과 멱등성 키(idempotency key)를 조합한 방식을 도입했습니다.

```java
@Service
public class PaymentService {
    private final RedissonClient redisson;

    public PaymentResult processPayment(PaymentRequest request) {
        String lockKey = "payment:lock:" + request.getIdempotencyKey();
        RLock lock = redisson.getLock(lockKey);

        try {
            if (!lock.tryLock(3, 10, TimeUnit.SECONDS)) {
                throw new PaymentConflictException("Duplicate payment request");
            }
            return doProcess(request);
        } finally {
            if (lock.isHeldByCurrentThread()) lock.unlock();
        }
    }
}
```

**결과**: 이중 결제 발생 건수 0건 (도입 후 8개월), 락 경합으로 인한 타임아웃 95% 감소
