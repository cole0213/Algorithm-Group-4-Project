<a name="프로젝트"></a>
## 3. 프로젝트

### 3-1. 팀플로우 (TeamFlow)

> 소규모 팀을 위한 경량 프로젝트 관리 도구

![TeamFlow 메인 화면](https://images.unsplash.com/photo-1611224923853-80b023f02d71?w=800)

![TeamFlow 칸반 보드](https://images.unsplash.com/photo-1557804506-669a67965ba0?w=800)

현재 다니는 회사에서 내부 도구로 쓰려고 만들었는데 생각보다 잘 써서 외부에 오픈도 했습니다.

**배경**
기존에 Notion + 스프레드시트 조합으로 프로젝트 관리하고 있었는데 너무 번거롭더라고요. 작은 팀에 맞는 가볍고 빠른 도구가 필요했습니다.

**구현 내용**
- 칸반 보드 (드래그 앤 드롭, react-beautiful-dnd)
- 실시간 협업 (Socket.io)
- 마크다운 에디터로 태스크 설명 작성
- 팀 멤버 초대 및 권한 관리

**기술 스택**
- Frontend: React, TypeScript, Tailwind CSS
- Backend: Node.js, Express, Socket.io
- DB: PostgreSQL (Supabase)
- 배포: Vercel (프론트), Railway (백엔드)

**링크**: [GitHub](https://github.com/dohyun-fullstack/teamflow) | [서비스](https://teamflow.app)

**현황**: MAU 약 80명 (지인 위주), 기능 요청 이슈 꾸준히 들어오는 중

### 3-3. 식물 물주기 알리미

졸업 프로젝트로 만든 IoT + 웹 연동 시스템이에요.

라즈베리파이 + 토양 수분 센서로 화분 상태를 모니터링하고, 수분이 부족하면 카카오톡 알림을 보내는 시스템입니다.

**구성**
- 라즈베리파이 4B + 토양 수분 센서 3개
- Python 수집 에이전트 (MQTT 프로토콜)
- Node.js 백엔드 (데이터 수신, 알림 로직)
- React 대시보드 (실시간 그래프)

**배운 것**
IoT 쪽은 처음이라 MQTT 브로커 설정하는 데 이틀 걸렸어요. 웹 개발이랑 다르게 하드웨어 디버깅이 생각보다 어렵다는 걸 알았습니다.
