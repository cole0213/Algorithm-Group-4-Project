## 프로젝트 / 연구

### API 보안 테스트 자동화 프레임워크 (사내 프로젝트)

배경: 수동 API 진단이 너무 반복적이고 범위가 방대해서 자동화가 필요했습니다.

Swagger/OpenAPI 스펙을 읽어서 자동으로 인증 관련 테스트 케이스를 생성하고 실행하는 도구.
BOLA, 수평적 권한 상승, 인증 토큰 없이 접근 가능 여부 등을 자동으로 검사.

```python
class AuthBypassTester:
    def __init__(self, spec_path: str, base_url: str):
        self.spec = load_openapi(spec_path)
        self.base_url = base_url

    def test_unauthenticated_access(self) -> list[Finding]:
        findings = []
        for path, methods in self.spec["paths"].items():
            for method, detail in methods.items():
                if self._requires_auth(detail):
                    resp = requests.request(
                        method.upper(),
                        f"{self.base_url}{path}",
                        # 인증 헤더 없이 요청
                    )
                    if resp.status_code not in (401, 403):
                        findings.append(Finding(
                            severity="HIGH",
                            title=f"Unauthenticated access: {method.upper()} {path}",
                            response=resp
                        ))
        return findings
```

현재 버즈페이 CI 파이프라인에 통합. 신규 API 개발 시 자동으로 기본 인증 테스트 수행.

### Semgrep 커스텀 룰 작성 (공개)

회사에서 만든 Semgrep 커스텀 룰 중 일반적인 것들을 공개 정리.
Java/Python 주요 취약 패턴 커버.

GitHub: github.com/jihoon-sec/semgrep-rules

주요 룰:
- `java.sqli.string-concat` : 문자열 연결로 SQL 쿼리 구성 탐지
- `python.xxe.lxml-unsafe` : lxml 파싱 시 external entity 미비활성화 탐지
- `js.jwt.none-algorithm` : JWT none 알고리즘 수용 탐지
