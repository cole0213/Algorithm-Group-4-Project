## 6. API 명세

### 기본 URL
- Backend: `http://localhost:8000`
- Frontend → Backend: Vite 프록시 `/api` → `http://localhost:8000/api`

### 엔드포인트 목록

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/portfolios` | 전체 포트폴리오 목록 반환 |
| POST | `/api/portfolios/add` | 포트폴리오 추가 (파일 또는 텍스트) |
| GET | `/api/portfolios/{id}/raw` | 원본 텍스트 반환 |
| GET | `/api/portfolios/export` | 전체 포트폴리오 JSON 내보내기 |
| POST | `/api/portfolios/import` | JSON 파일로 포트폴리오 가져오기 (병합) |
| DELETE | `/api/portfolios/{id}` | 포트폴리오 삭제 |
| POST | `/api/analyze` | 필요 스펙 매칭 분석 + 정렬 |
| GET | `/api/search` | 키워드 검색 (cross / intra 모드) |
| GET | `/api/similar` | 유사 문장 검출 |

### 포트폴리오 추가 (`POST /api/portfolios/add`)

**요청 (multipart/form-data)**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `file` | File | 둘 중 하나 | `.pdf` / `.md` / `.txt` |
| `text` | string | 둘 중 하나 | 직접 붙여넣은 텍스트 |
| `name` | string | 선택 | 지원자 이름 |
| `position` | string | 선택 | `general` \| `frontend` \| `backend` \| `data` (기본: `general`) |

**응답**
```json
{
  "message": "추가 완료",
  "portfolio": { ... },
  "solar": {
    "used": true,
    "elapsed": 3.2,
    "tokens": { "prompt_tokens": 800, "completion_tokens": 300 },
    "truncated": false
  }
}
```

### 분석 (`POST /api/analyze`)

**요청**
```json
{
  "required_specs": ["React", "Python", "Docker"],
  "sort_key": "match"
}
```
`sort_key`: `"match"` | `"career"` | `"name"`

**응답**: 지원자별 `match_score`, `skills_match`, `matched_skills` 포함 목록

### 검색 (`GET /api/search`)

| 파라미터 | 설명 |
|----------|------|
| `q` | 검색 키워드 |
| `mode` | `cross` (전체 지원자) \| `intra` (포트폴리오 내부) |
| `portfolio_id` | intra 모드 시 대상 포트폴리오 ID |

### 유사 문장 (`GET /api/similar`)

| 파라미터 | 설명 |
|----------|------|
| `ids` | 쉼표 구분 포트폴리오 ID (생략 시 전체) |

## 8. Solar LLM 통합

### 처리 흐름

```
업로드 (PDF / MD / TXT / 텍스트 직접 입력)
  → raw_text 추출
  → 8,000자 초과 시 _truncated = True 플래그 설정 (절단 처리)
  → Solar LLM 파싱 (position 파라미터로 직군별 프롬프트 분기)
  → 실패 시: basic_parser 폴백
  → _normalize() — 타입 강제·스키마 정제
  → session.json 자동 저장
```

### 직군별 포지션 (`position` 파라미터)

| 값 | 설명 |
|----|------|
| `general` | 추가 지침 없음 (기본) |
| `frontend` | CSS 프레임워크·번들러·상태관리 라이브러리 추출 강조 |
| `backend` | DB·ORM·서버 프레임워크·인프라 도구 추출 강조 |
| `data` | ML·DL 프레임워크·데이터 파이프라인·논문/대회 실적 추출 강조 |

### 환각 방지 원칙

- 원문에서 찾을 수 없는 내용은 빈값으로 — 생성·추론 금지
- `intro`, `desc`: 원문 텍스트 그대로 복사 (요약·의역·수치 추가 금지)
- `career_years`: 원문에 명시된 정수만. 날짜 계산 금지. 없으면 0
- `skills`: 보유·사용 기술로 명확히 언급된 경우에만 포함
- 부정 지시보다 긍정 지시 우선 ("~인 경우에만 포함하십시오")
- 3가지 반례(few-shot) 포함: 배우고 싶다 → skills 미포함, 수치 생성 금지, 날짜 계산 금지

### 컴포넌트별 기능

#### TopBar
- 키워드 검색 (300ms debounce, 별칭 통합 검색)
- 필요 스펙 입력 → 분석 버튼 (Enter 키 지원)
- 정렬 기준 선택 (매칭률 높은 순 / 경력 많은 순 / 이름 순)
- `+ 추가` — UploadModal 열기
- `↓ 내보내기` — 현재 포트폴리오 전체 JSON 다운로드
- `↑ 불러오기` — JSON 파일 선택 → 기존 목록에 병합
- `⚙ 설정` — SettingsDrawer 열기

#### Sidebar
- 지원자 목록 (수직 스크롤)
- 항목 클릭 → 우측에 포트폴리오 패널 추가 (최대 4개)
- 매칭률 배지 (색상: 70%↑ 초록 / 40~69% 주황 / 39%↓ 빨강)
- ✕ 버튼 → 해당 포트폴리오 삭제

#### PortfolioPanel
- 패널 상단 탭: 이름 + 매칭률 배지 + AI 레이블 + 유사문장 숨기기 버튼 + ✕ 닫기
- AI 레이블: Solar LLM으로 파싱된 경우 `AI` 표시 (세션 재로드 후에도 유지)
- ⚠ 절단 경고: 원문이 8,000자 초과 시 상단 경고 배너
- 패널 내 검색 바 (intra): 실시간 하이라이트 + 결과 건수 + 첫 결과 자동 스크롤
- 본문: 기본 정보 / 기술 스택 / 자기소개 / 프로젝트 / 수상 및 활동
- 텍스트 렌더링 레이어: 유사문장 하이라이트 → 전역 검색어 → 패널 내 검색어 순 적용

## 10. 기능 명세

### 포트폴리오 추가

1. UploadModal에서 포지션 선택 → 파일 또는 텍스트 입력
2. POST `/api/portfolios/add` 호출
3. Solar LLM 파싱 (실패 시 basic_parser 폴백)
4. session.json 자동 저장
5. 사이드바 목록 갱신

### 스펙 매칭 분석

1. 필요 스펙 입력 (쉼표 구분) → `분석` 클릭
2. POST `/api/analyze` 호출
3. 해시 테이블 O(1) 스킬 매칭 여부 확인
4. LCS DP로 매칭 점수 산출 (0~100%)
5. 선택한 기준으로 정렬
6. 결과 → 사이드바 매칭률 배지, 패널 스킬 하이라이트 업데이트

### 키워드 검색 (전체 — cross mode)

1. 상단 검색창 입력 (300ms debounce)
2. 별칭 해시맵에서 동의어 목록 조회 (O(1))
3. Edit Distance로 오타 허용 확장 매칭
4. BST 인덱스 검색과 합집합
5. 미보유 지원자 흐리게 처리 (opacity 0.4)

### 패널 내 검색 (intra mode)

1. 각 패널 상단 검색 바에 입력 (300ms debounce)
2. BST 알고리즘 검증 호출 (알고리즘 시연)
3. 화면 표시 텍스트 기준 정규식 매칭 → 결과 건수 표시
4. `.intra-highlight` 첫 번째 요소로 자동 스크롤

### 유사 문장 검출

1. 설정 → `유사 문장 표시` ON
2. 감지 범위 선택 (전체 / 열린 패널)
3. GET `/api/similar?ids=...` 호출
4. Rabin-Karp 슬라이딩 윈도우로 후보 구간 탐색
5. LCS로 유사도 측정 → 임계값 초과 시 같은 그룹 색상 부여
6. 패널에서 그룹별 색상 하이라이트 표시
7. 숨기기 ON → opacity 0.3 처리 (완전 삭제 아님)
8. 패널별 독립 숨기기/보이기 버튼 제공

### 동기화 스크롤

- 비율 기반 스크롤 동기화 (각 패널의 최대 스크롤 높이 대비 비율 계산)
- 툴바 토글로 ON/OFF 즉시 전환

### 데이터 영속성

- 추가·삭제·임포트 시 `backend/session.json` 자동 저장
- 서버 재시작 시 session.json 로드 (없으면 빈 목록으로 시작)
- export → JSON 파일 다운로드 / import → 기존 목록에 병합 (중복 ID 자동 처리)

## 12. 실행 방법

```bash
# 1. 백엔드 실행
cd backend
python -m venv venv
venv\Scripts\activate          # Windows
pip install -r requirements.txt
uvicorn main:app --reload      # http://localhost:8000

# 2. 프론트엔드 실행 (별도 터미널)
cd frontend
npm install
npm run dev                    # http://localhost:5173
```
