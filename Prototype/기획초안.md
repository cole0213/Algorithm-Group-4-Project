# 포트폴리오 비교 리뷰어 - 프로젝트 계획서 (v2)

## 주제

면접 서류심사의 편의성을 위한, 개발자 지원자의 포트폴리오 모음을 정형화된 서식으로 정리해 나열하여 보여주는 사이트

---

## 사용자 입력

| 구분 | 항목                                     |
| ---- | ---------------------------------------- |
| 필수 | 지원자 포트폴리오 다수 (PDF / MD)        |
| 필수 | 정형화된 서식 (어떤 항목으로 정리할지)   |
| 필수 | 필요 스펙 (요구 기술 스택, 경력 조건 등) |

---

## 출력

| 항목        | 설명                                                   |
| ----------- | ------------------------------------------------------ |
| 정형화 결과 | 포트폴리오가 지정된 서식으로 자동 정리됨               |
| 수평 나열   | 정리된 포트폴리오가 사이트 UI에 가로로 나열되어 표시됨 |
| 하이라이트  | 필요 스펙에 해당하는 항목이 강조되어 표시됨            |

---

## 전체 흐름

```
사용자 입력 (포트폴리오 + 서식 + 필요 스펙)
      ↓
① Solar: 포트폴리오에서 정보 추출
         (이름, 기술스택, 경력, 프로젝트, 학력 등)
      ↓
② 해시 테이블: 필요 스펙 키워드 저장 및 O(1) 매칭 탐지
      ↓
③ 별칭 해시맵 + Edit Distance (DP): 기술명 정규화 및 통합 매칭
   (예: "ReactJS" ↔ "React" / "파이선" ↔ "Python" 등 표기 차이 및 오타 흡수)
      ↓
④ LCS (DP): 필요 스펙 vs 지원자 스펙 공통 항목 계산
   → 매칭 점수 산출
      ↓
⑤ 정렬: 매칭 점수 기준 지원자 순위 정렬
      ↓
⑥ BST: 특정 키워드로 지원자 빠르게 검색/필터링
      ↓
⑦ 유사 문장 검출: Rabin-Karp / LCS로 포트폴리오 간
   유사 문장 감지 → 색상 부여
      ↓
결과 화면 표시
(정형화된 서식으로 수평 나열 + 하이라이트 + 원본 링크 버튼)
```

## 알고리즘별 예시

### ① Solar
```
입력: 지원자 포트폴리오 PDF / GitHub URL / 개인 사이트 URL

출력 (정형화된 서식):
{
  "이름": "홍길동",
  "기술스택": ["React", "Python", "Docker", "AWS"],
  "경력": "3년",
  "프로젝트": [
    {"이름": "쇼핑몰 플랫폼", "기간": "6개월", "역할": "프론트엔드 개발"},
    {"이름": "챗봇 서비스", "기간": "3개월", "역할": "백엔드 개발"}
  ],
  "학력": "OO대학교 컴퓨터공학과",
  "포트폴리오링크": "https://github.com/honggildong"
}
```

### ② 해시 테이블 (Hash Table)
```
필요 스펙: ["React", "Python", "Docker"]

해시셋 구성 (O(1) 조회):
requiredSpecs = {"React", "Python", "Docker"}

지원자A 기술스택: ["React", "Vue", "Python", "Git"]

매칭 확인:
"React"  → requiredSpecs에 있음 → O(1) → 하이라이트
"Python" → requiredSpecs에 있음 → O(1) → 하이라이트
```

### ③ 기술명 정규화 및 통합 검색 (별칭 해시맵 + Edit Distance)
```
문제: 기술명이 사람마다 다르게 표기되며(ReactJS vs React.js), 오타도 발생함(파이선)

별칭 해시맵 (사전 정의):
aliasMap = {
  "python": ["py", "파이썬", "python", ".py"],
  "react": ["react", "reactjs", "react.js"],
  "javascript": ["js", "자바스크립트", "javascript", ".js"]
}

[ 매칭 및 검색 과정 ]
필요 스펙 또는 검색어: "Python" / "React"

① aliasMap에서 동의어 목록 조회 (O(1))
   → "Python" ➔ ["py", "파이썬", "python", ".py"]
② 각 별칭에 대해 지원자 텍스트와 Edit Distance 비교 (임계값 이하 허용)
   → 지원자A "파이선" ➔ "파이썬"과 거리 1 (매칭 O)
   → 지원자B "ReactJS" ➔ "react" 그룹과 매칭 O
③ 포트폴리오 텍스트뿐만 아니라 코드블럭 언어 태그(```python)도 동일 기준으로 동시 탐색

→ 필요 스펙 매칭(하이라이트)과 사용자의 키워드 검색 로직을 일원화하여 표기법이 달라도 한 번에 탐지
```

### ④ LCS (DP)
```
필요 스펙:  [React, Python, Docker, AWS, Git]
지원자A 스펙: [React, Vue, Python, Git, Linux]

LCS 계산:
공통: React, Python, Git (길이 3)

매칭 점수 = 공통 항목 수 / 필요 스펙 수 = 3 / 5 = 60%
```

### ⑤ 정렬
```
매칭 점수 높은 순 / 경력 많은 순 등 인사담당자가 원하는 기준으로 정렬 전환 가능
```

### ⑥ BST
```
[ A. 지원자 전체 검색 ] 키워드 기준 지원자 인덱스 구성 (O(log n))
[ B. 포트폴리오 내부 검색 ] 포트폴리오 내 토큰 단위 BST 구성으로 키워드 위치/맥락 탐색
```

### ⑦ 유사 문장 검출 (Rabin-Karp / LCS)
```
포트폴리오 간 유사 문장 탐지:
① Rabin-Karp: 슬라이딩 윈도우로 해시값 비교 → 유사 구간 빠르게 탐지
② LCS: 유사 문장의 공통 부분 길이 측정 → 유사도 수치화
③ 임계값 초과 시 동일 색상 부여 및 [숨기기/보이기] 기능 제공
```

## 기능 목록

| 기능               | 설명                                                                        |
| ------------------ | --------------------------------------------------------------------------- |
| 수평 나열          | 지원자 카드를 가로로 나란히 표시                                            |
| 동기화 스크롤      | 우측 포트폴리오 패널들끼리 스크롤 연동. ON/OFF 토글 가능                    |
| 하이라이트         | 필요 스펙에 해당하는 항목 강조 표시                                         |
| 원본 링크          | 실제 포트폴리오로 이동하는 버튼                                             |
| 정렬 전환          | 매칭률 / 경력 / 이름 등 기준 변경                                           |
| 키워드 검색 (전체) | 특정 기술 보유 지원자만 필터링 — 지원자 전체 대상 BST 탐색                  |
| 키워드 검색 (내부) | 개별 포트폴리오 내 키워드 위치 탐색 — 발견 위치 하이라이트 및 스크롤        |
| 유연한 통합 검색   | 오타 및 별칭(py, 파이선 등)을 Edit Distance와 해시맵으로 보정하여 통합 검색 |
| 유사 문장 검출     | 포트폴리오 간 유사 문장에 색상 부여, 버튼으로 숨기기/보이기 전환            |
| 숨김 표시          | 유사 문장 숨길 시 완전히 지우지 않고 옅은 텍스트로 존재 표시                |
| 기능 ON/OFF        | 모든 기능 개별 토글 가능                                                    |
