# 포트폴리오 비교 리뷰어 - 프로젝트 계획서

## 주제

면접 서류심사의 편의성을 위한, 개발자 지원자의 포트폴리오 모음을 정형화된 서식으로 정리해 나열하여 보여주는 사이트

---

## 사용자 입력

| 구분 | 항목                                     |
| ---- | ---------------------------------------- |
| 필수 | 지원자 포트폴리오 다수 (PDF / URL)       |
| 필수 | 정형화된 서식 (어떤 항목으로 정리할지)   |
| 필수 | 필요 스펙 (요구 기술 스택, 경력 조건 등) |

---

## 출력

| 항목 | 설명 |
| ---- | ---- |
| 정형화 결과 | 포트폴리오가 지정된 서식으로 자동 정리됨 |
| 수평 나열 | 정리된 포트폴리오가 사이트 UI에 가로로 나열되어 표시됨 |
| 하이라이트 | 필요 스펙에 해당하는 항목이 강조되어 표시됨 |

---

## 전체 흐름

```
사용자 입력 (포트폴리오 + 서식 + 필요 스펙)
      ↓
① Solar: 포트폴리오에서 정보 추출
         (이름, 기술스택, 경력, 프로젝트, 학력 등)
      ↓
② 해시 테이블: 필요 스펙 키워드 저장 및 O(1) 매칭 탐지
      ↓
③ Edit Distance (DP): 기술 스택 유사도 계산
   (예: "ReactJS" ↔ "React.js" → 같은 기술로 처리)
      ↓
④ LCS (DP): 필요 스펙 vs 지원자 스펙 공통 항목 계산
   → 매칭 점수 산출
      ↓
⑤ 정렬: 매칭 점수 기준 지원자 순위 정렬
      ↓
⑥ BST: 특정 키워드로 지원자 빠르게 검색/필터링
      ↓
⑦ 유사 언어 동시검색: 별칭 해시맵 + Edit Distance로
   "Python" → py, 파이썬, python 코드블럭 동시 탐색
      ↓
⑧ 유사 문장 검출: Rabin-Karp / LCS로 포트폴리오 간
   유사 문장 감지 → 색상 부여
      ↓
결과 화면 표시
(정형화된 서식으로 수평 나열 + 하이라이트 + 원본 링크 버튼)
```

## Solar LLM 특성

### 우위
| 항목 | 내용 |
| ---- | ---- |
| 한국어 자연스러움 | 한국 기업·도메인 데이터로 학습, 자연스러운 한국어 출력 |
| 비용/속도 | 31B 사이즈, 단일 GPU로 GPT-4급 일부 태스크 처리 |
| 문서 구조 이해 | HTML / Markdown / 표 인식이 동급 모델보다 우수 |
| 무료 API | 2026년 3월까지 무료 → 학생 프로젝트에 부담 없음 |

### 약점
| 항목 | 내용 |
| ---- | ---- |
| 일반 지식·복잡 추론 | GPT-5 / Claude / Gemini 최상위 모델 대비 명확히 열세 |
| 코드 생성 품질 | Claude Opus / GPT-5 대비 낮음 |
| 생태계 | LangChain / LlamaIndex 통합은 되지만 레퍼런스·튜토리얼 적음 |
| 멀티모달 | 이미지/오디오 입력 처리는 GPT-4o, Gemini가 압도적 |
| 컨텍스트 | 32k — Gemini 1M / Claude 200k 대비 짧음 |

### 프로젝트 적용 시 주의점
- 포트폴리오 1건당 텍스트가 32k 토큰 이내로 들어오도록 분할 처리 필요
- 코드 생성은 Solar에 맡기지 않고 파싱·추출 역할에만 집중
- 한국어 포트폴리오 처리에는 강점, 영문 위주 포트폴리오는 품질 검토 필요

## 알고리즘별 예시

### ① Solar
```
입력: 지원자 포트폴리오 PDF / GitHub URL / 개인 사이트 URL

출력 (정형화된 서식):
{
  "이름": "홍길동",
  "기술스택": ["React", "Python", "Docker", "AWS"],
  "경력": "3년",
  "프로젝트": [
    {"이름": "쇼핑몰 플랫폼", "기간": "6개월", "역할": "프론트엔드 개발"},
    {"이름": "챗봇 서비스", "기간": "3개월", "역할": "백엔드 개발"}
  ],
  "학력": "OO대학교 컴퓨터공학과",
  "포트폴리오링크": "https://github.com/honggildong"
}
```
→ 제각각인 포트폴리오를 통일된 서식으로 자동 정리

### ② 해시 테이블 (Hash Table)
```
필요 스펙: ["React", "Python", "Docker"]

해시셋 구성 (O(1) 조회):
requiredSpecs = {"React", "Python", "Docker"}

지원자A 기술스택: ["React", "Vue", "Python", "Git"]

매칭 확인:
"React"  → requiredSpecs에 있음 → O(1) → 하이라이트
"Vue"    → requiredSpecs에 없음 → O(1) → 표시 없음
"Python" → requiredSpecs에 있음 → O(1) → 하이라이트
"Git"    → requiredSpecs에 없음 → O(1) → 표시 없음

→ 지원자 수 / 스펙 수가 많아져도 매칭 속도 일정 (O(1))
→ UI에서 매칭된 키워드가 강조되어 표시됨
```

---

### ③ Edit Distance (DP)
```
문제: 기술명이 사람마다 다르게 표기됨

필요 스펙: "React"

지원자A: "React"   → 거리 0 → 매칭 O
지원자B: "ReactJS" → 거리 2 → 임계값 이하 → 매칭 O
지원자C: "React.js"→ 거리 3 → 임계값 이하 → 매칭 O
지원자D: "Angular" → 거리 7 → 임계값 초과 → 매칭 X

→ 표기 방식이 달라도 같은 기술로 인식 가능
```

---

### ④ LCS (DP)
```
필요 스펙:  [React, Python, Docker, AWS, Git]
지원자A 스펙: [React, Vue, Python, Git, Linux]

LCS 계산:
공통: React, Python, Git (길이 3)

매칭 점수 = 공통 항목 수 / 필요 스펙 수
           = 3 / 5 = 60%

→ 지원자별 매칭 점수 산출
→ 화면에 "매칭률 60%" 표시
```

---

### ⑤ 정렬
```
매칭 점수 높은 순:
지원자C (80%) → 지원자A (60%) → 지원자B (40%) → 지원자D (20%)

경력 많은 순:
지원자B (5년) → 지원자A (3년) → 지원자C (2년) → 지원자D (1년)

→ 인사담당자가 원하는 기준으로 정렬 전환 가능
```

---

### ⑥ BST
```
BST 검색은 두 가지 범위로 동작:

[ A. 지원자 전체 검색 (Cross-Portfolio) ]
  키워드를 기준으로 BST에 지원자 인덱스 구성
  "Docker" 입력 → BST 탐색 O(log n)
                → Docker 보유 지원자 목록 반환
                → 해당 지원자 카드만 필터링하여 표시

  "Python" 입력 → Python 보유 지원자만 즉시 조회
  → 지원자 수가 많아져도 빠른 검색 유지

[ B. 포트폴리오 내부 검색 (Intra-Portfolio) ]
  개별 포트폴리오의 텍스트 내용을 토큰 단위로 BST 구성
  "Docker" 입력 → 해당 포트폴리오 내에서 키워드 위치 탐색
                → 발견된 위치 하이라이트 표시
                → "Docker" 언급된 단락으로 스크롤 이동

  → 포트폴리오 내 특정 키워드가 어떤 맥락에서 쓰였는지 확인 가능

[ 검색 범위 전환 ]
  UI 토글로 "전체 검색 / 내부 검색" 모드 전환 가능
  전체 검색: 보유 여부 기준 지원자 필터링
  내부 검색: 카드 내 키워드 위치 하이라이트
```

---

### ⑦ 유사 언어 동시검색 (별칭 해시맵 + Edit Distance)
```
별칭 해시맵 (사전 정의):
aliasMap = {
  "python": ["py", "파이썬", "python", ".py"],
  "javascript": ["js", "자바스크립트", "javascript", ".js"],
  "typescript": ["ts", "타입스크립트", "typescript", ".ts"],
  ...
}

"Python" 검색 시:
① aliasMap에서 동의어 목록 조회 (O(1))
   → ["py", "파이썬", "python", ".py"]
② 각 별칭에 대해 Edit Distance로 유사 표기도 포함
   ("파이썬" → "파이선" 같은 오타도 허용)
③ 포트폴리오 전체에서 위 목록과 매칭되는 항목 탐색

→ 표기법이 달라도 같은 언어/기술을 한 번의 검색으로 탐지
→ 코드블럭 내 언어 태그(```python)도 함께 인식
```

---

### ⑧ 유사 문장 검출 (Rabin-Karp / LCS)
```
포트폴리오 간 유사 문장 탐지:

지원자A 자기소개: "저는 사용자 경험을 중시하는 프론트엔드 개발자입니다."
지원자B 자기소개: "저는 사용자 경험을 중요시하는 프론트엔드 개발자입니다."

① Rabin-Karp: 슬라이딩 윈도우로 해시값 비교 → 유사 구간 빠르게 탐지
② LCS: 유사 문장의 공통 부분 길이 측정 → 유사도 수치화
③ 임계값 초과 시 동일 색상 부여 (지원자 쌍마다 다른 색)

표시 방식:
- 유사 문장 구간에 색상 하이라이트
- [숨기기] 버튼 → 해당 구간이 옅은 텍스트로 변경 (내용은 유지)
- [보이기] 버튼 → 원래 색상 하이라이트로 복원

→ 포트폴리오 표절/중복 문구 빠르게 파악 가능
```

## 화면 구성 (UI 개요)

### 레이아웃 구조

```
┌──────────────────────────────────────────────────────────────────┐
│  🔍 [검색창 (유사어 포함)]   [정렬 기준 ▾]   [⚙ 설정]           │ ← 상단 바
├────────────────┬─────────────────────────────────────────────────┤
│  지원자 리스트  │  포트폴리오 영역        [동기화 스크롤 ●/○]     │
│                │                                                  │
│  지원자C  80%  │  지원자C · 김지수  80% [X]  │  지원자A · 이민준  │
│  지원자A  60%  │ ─────────────────────────── │ ─────────────────  │
│  지원자B  40%  │  # 김지수 포트폴리오        │  # 이민준 포트폴리오│
│  지원자D  20%  │                             │                    │
│                │  ## 기본 정보               │  ## 기본 정보      │
│  ↑ 수직 스크롤 │  경력: 2년 / 학력: OO대    │  경력: 3년 / ...   │
│                │                             │                    │
│                │  ## 기술 스택               │  ## 기술 스택      │
│                │  [React] [Python] Vue       │  [React] [Python]  │
│                │                             │                    │
│                │  ## 자기소개                │  ## 자기소개       │
│                │  ~~저는 사용자 중심의~~     │  ~~저는 사용자~~   │
│                │                             │                    │
│                │  원본 포트폴리오 보기 →     │  원본 포트폴리오 → │
│                │                             │                    │
│                │ ←── 패널 최대 4개, 동적 너비 분할 (flex: 1) ──→  │
└────────────────┴──────────────────────────────────────────────────┘
```
`[]` = 필요스펙 하이라이트 / `~~텍스트~~` = 유사문장 색상 표시
패널 구분: 세로선 (border-right) · 패널 탭 하단에 지원자별 색상 언더라인

### 영역별 역할

| 영역 | 역할 |
| ---- | ---- |
| 상단 바 | 검색창 + 정렬 드롭다운 + 설정 버튼만 고정 노출 |
| 좌측 리스트 | 지원자 이름 + 매칭률 수직 목록. 클릭 시 우측에 패널 추가. 독립 스크롤 |
| 우측 포트폴리오 영역 | 선택된 지원자 패널을 수평 나열. 최대 4개, 동적 너비 분할 (1개=100%, 2개=50%, 3개=33%, 4개=25%) |
| 설정 드로어 | ⚙ 버튼으로 열리는 사이드 패널. 모든 기능 ON/OFF 토글 모음 |

### 스크롤 동작

## 기능 목록

| 기능               | 설명                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| 수평 나열          | 지원자 카드를 가로로 나란히 표시                                     |
| 동기화 스크롤      | 우측 포트폴리오 패널들끼리 스크롤 연동 (좌측 리스트는 제외). ON/OFF 토글 가능 |
