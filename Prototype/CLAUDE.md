# CLAUDE.md

## 프로젝트 개요

면접 서류심사 편의를 위한 개발자 포트폴리오 정형화·비교 웹 사이트.
지원자 포트폴리오(PDF/URL)를 Solar LLM으로 파싱해 정형화된 서식으로 나열하고,
필요 스펙 하이라이트·유사문장 검출·키워드 검색 등을 제공한다.

## 문서 구조

| 파일 | 내용 |
| ---- | ---- |
| `기획초안.md` | 주제, 입출력, 전체 흐름, 알고리즘별 설명, UI 구성, 기능 목록 |
| `개발환경.md` | 기술 스택, 디렉토리 구조, 설치·실행 명령어, 환경 변수 |
| `style.md` | 색상 팔레트, 타이포그래피, 레이아웃, 컴포넌트, 인터랙션 기준 |

## 기술 스택

- **Frontend**: React + Vite
- **Backend**: Python + FastAPI
- **LLM**: Upstage Solar API
- **PDF 파싱**: pdfplumber

## 알고리즘 목록

| # | 알고리즘 | 역할 |
|---|----------|------|
| 1 | Solar LLM | 포트폴리오 파싱·정형화 |
| 2 | 해시 테이블 | 필요 스펙 O(1) 매칭·하이라이트 |
| 3 | Edit Distance (DP) | 기술명 표기 차이 흡수 |
| 4 | LCS (DP) | 매칭 점수 산출 |
| 5 | 정렬 | 매칭률·경력 기준 순위 정렬 |
| 6 | BST | 키워드 검색 (전체/내부 두 모드) |
| 7 | 별칭 해시맵 + Edit Distance | 유사 언어명 동시검색 |
| 8 | Rabin-Karp + LCS | 포트폴리오 간 유사 문장 검출 |

## UI 구조 요약

```
상단 바 (검색 + 정렬 + ⚙ 설정)
├── 좌측: 지원자 리스트 (수직, 독립 스크롤)
└── 우측: 포트폴리오 패널 최대 4개 수평 나열
          (동기화 스크롤 ON/OFF 토글)
          설정은 ⚙ 드로어로 분리
```

## 작업 규칙

- 기획초안.md는 기획 문서이며 코드가 아님. 내용 변경 시 사용자 승인 후 반영
- style.md 기준을 벗어나는 색상·폰트·여백 사용 금지
- 알고리즘은 `backend/services/algorithms/` 하위에 파일별로 분리 구현
- Solar API 키는 `.env`에서만 관리, 코드에 하드코딩 금지
- 포트폴리오 1건당 32k 토큰 초과 시 분할 처리 필요 (Solar 컨텍스트 제한)
